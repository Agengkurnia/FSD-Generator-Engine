1. PENDAHULUAN 
1.1 Tujuan Dokumen 
Dokumen ini menjelaskan spesifikasi fungsional dari sistem New RM Sample Management yang merupakan 
revamp dari tiga modul terpisah (Document Sample, Sample Purpose, dan RM Evaluation) menjadi satu sistem 
terintegrasi dengan 4 tahapan proses. 
1.2 Ruang Lingkup 
Sistem ini mencakup: - Penerimaan dan dokumentasi sample RM dari supplier - Pendefinisian tujuan penggunaan 
sample - Evaluasi teknis dan pengujian laboratorium - Keputusan disposisi akhir (Approve/Reject/On Hold) 
1.3 Stakeholder 
• 
R&D Department: Pengguna utama untuk evaluasi sample 
• 
QA/QC Department: Reviewer hasil pengujian 
• 
Purchasing Department: Monitoring status sample untuk procurement 
• 
Management: Approval keputusan akhir 
 
2. RINGKASAN BUSINESS FLOW 
2.1 Proses As-Is (Legacy System) 
Sistem lama menggunakan 3 form terpisah: 1. Document Sample: Input data penerimaan sample 2. Sample 
Purpose: Definisi tujuan penggunaan 3. RM Evaluation: Evaluasi teknis dan hasil lab 
Masalah: - Data terpisah-pisah - Duplikasi input - Tracking progress sulit - Tidak ada workflow approval terintegrasi 
2.2 Proses To-Be (New System) 
2.2.1 Simplified Flow Diagram 
 

2.2.2 Detailed Workflow Flowchart 


Keuntungan: - Single entry point - Progress tracking real-time - Workflow approval terintegrasi - Data consistency 
terjaga

4. STRUKTUR DATABASE 
4.1 Konvensi Penamaan 
[!IMPORTANT] PK/FK Convention yang disepakati: - PK di tabel sendiri → intId (INT IDENTITY, auto-
increment) + txtId (VARCHAR(50), GUID) - FK di tabel lain → nama descriptive, 
e.g. intSampleHeaderId merujuk ke trRMSample_Header.intId (hanya intId, tanpa txtId) - Table 
names: tr{ModuleName}_{Type} (contoh: trRMSample_Header) - Column names: Hungarian 
notation dengan prefix int, txt, dec, dt, dtm, bit - Audit columns: txtCreatedBy, dtmCreatedDate, 
txtUpdatedBy, dtmUpdatedDate, bitActive - Denormalized columns: LOV names disimpan bersama 
ID-nya (e.g. intSupplierId + txtSupplierName) - Database: SQL Server (IDC_Formulation) - DDL Script: 
idc-system/Database/Scripts/03_Step1_DocumentSample_DDL.sql

5. BUSINESS RULES 
5.1 General Rules 
1. Sample No auto-generated dengan format: SMPL-YYYY-XXXX 
2. Setiap step harus diselesaikan sebelum lanjut ke step berikutnya 
3. Data dapat disimpan sebagai draft di setiap step 

4. User dapat kembali ke step sebelumnya untuk edit (jika status masih Draft) 
5. PIC auto-filled dari user yang login, tidak dapat diubah 
5.2 Step-Specific Rules 
Step 1 (Document): - Minimal 1 dokumen COA harus diupload - Jika Halal Category dipilih, Halal Body wajib diisi - 
Allergen minimal 1 harus dipilih - Currency dipilih dari 20+ mata uang yang tersedia (AUD, CHF, CNY, EUR, IDR, dll) - 
Type Document hanya 3 pilihan: RM, RM(BTP), Vitamin & Mineral 
Step 2 (Purpose): - Jika Sample Purpose Type = “New Ingredients”, Item Code Existing disabled - Minimal 1 product 
harus didefinisikan di Apply To Product - Objective wajib diisi 
Step 3 (Evaluation): - Setiap parameter harus memiliki minimal COA Result atau Analysis Result - Jika nilai di luar 
range regulasi, warning ditampilkan - Overall Score dihitung otomatis dari semua parameter - User dapat add COA 
Result dan Analysis Result secara terpisah - User dapat view gabungan COA dan Analysis Result dalam satu modal 
Step 4 (Disposition): - Jika Decision = “Rejected” atau “On Hold”, Reason wajib diisi - Jika Approval Required = Yes, 
Approver wajib dipilih - Jika Decision = “On Hold”, user dapat kembali ke Step 3 untuk tambah evaluasi 
5.3 Workflow Rules 
• 
Draft: User dapat edit semua field, tombol “Save Draft” aktif di semua step 
• 
Pending Approval: Hanya approver yang dapat approve/reject 
• 
Approved: Data read-only, tidak dapat diedit 
• 
Rejected: Data read-only 
• 
On Hold: User dapat edit Step 3 saja 
5.4 Navigation Rules 
• 
Tombol “Previous” disabled di Step 1 
• 
Tombol “Next” disabled di Step 4 
• 
Tombol “Submit Complete” hanya muncul di Step 4 
• 
Progress indicator menunjukkan step aktif dengan highlight
