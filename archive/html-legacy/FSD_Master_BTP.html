1. INTRODUCTION

1.1 Purpose

Dokumen ini menjelaskan spesifikasi fungsional untuk module Master BTP (Bahan Tambahan Pangan) dalam sistem IDC (Integrated Data Center) untuk RM Selection. Module ini dirancang untuk mengelola data master Bahan Tambahan Pangan sesuai dengan regulasi BPOM RI.

1.2 Scope

Dokumen ini mencakup 3 sub-module utama:

  • Master BTP - Pengelolaan data Bahan Tambahan Pangan
  • Master BTP Function - Pengelolaan fungsi-fungsi BTP
  • BTP Function Mapping - Mapping relasi BTP dengan fungsinya

1.3 Definitions and Acronyms

Term Definition
BTP Bahan Tambahan Pangan - Food Additive
BPOM Badan Pengawas Obat dan Makanan
IDC Integrated Data Center
CRUD Create, Read, Update, Delete
LOV List of Values

2.3 Menu BTP Function Mapping

Menu ini digunakan untuk mapping relasi many-to-many antara BTP dan Fungsinya. Satu BTP dapat memiliki multiple fungsi.

2.3.1 Layout BTP Function Mapping

Screenshot:

Figure: BTP Function Mapping Page - Mapping BTP dengan Fungsi

Fungsi Utama:

  • Menampilkan header BTP (readonly)
  • Menampilkan tabel fungsi yang sudah di-assign ke BTP
  • Menambah fungsi baru melalui LOV modal
  • Menghapus fungsi dari BTP
  • Bulk save semua mapping

Section 1: BTP Header (Read-Only)

Field Description
BTP Code Kode BTP (readonly)
BTP Name Nama BTP (readonly)
Active Status BTP (readonly checkbox)

Section 2: Function Mapping Table

Column Type Description
Action Button Delete button untuk remove fungsi
Function Code LOV Kode fungsi dengan search button
Function Name Text (readonly) Nama fungsi (auto-filled dari LOV)
Active Checkbox Status aktif mapping

Features:

  • Add Function Button: Menambah row baru di top table
  • Search Function Modal: LOV popup untuk select fungsi
  • Delete Function: Remove row dari table (instant, no confirmation)
  • Save Button: Bulk save semua mappings (transaction: all or nothing)
  • Back Button: Kembali ke BTP Index

Validation Rules:

  • Tidak boleh duplicate fungsi dalam satu BTP
  • Minimal harus ada 1 fungsi untuk save
  • Function Code dan Name wajib diisi (via search)
  • Konfirmasi sebelum save: "Are you sure to save all mappings?"

3. DATABASE SPECIFICATIONS

3.1 Table: M_BTP

Purpose: Menyimpan master data Bahan Tambahan Pangan

Column Name Data Type Nullable Default Description
intBtpId INT No Auto Primary Key
txtBtpCode VARCHAR(20) No - Kode BTP (INS Number)
txtBtpName VARCHAR(200) No - Nama BTP
bitActive BIT No 1 Status aktif
txtCreatedBy VARCHAR(50) No - User pembuat
dtCreatedDate DATETIME No GETDATE() Tanggal dibuat
txtModifiedBy VARCHAR(50) Yes NULL User terakhir modifikasi
dtModifiedDate DATETIME Yes NULL Tanggal terakhir modifikasi

Indexes:

  • PRIMARY KEY: intBtpId
  • UNIQUE INDEX: txtBtpCode
  • INDEX: bitActive

3.2 Table: M_BTP_FUNCTION

Purpose: Menyimpan master fungsi BTP

Column Name Data Type Nullable Default Description
intBtpFunctionId INT No Auto Primary Key
txtFunctionCode VARCHAR(10) No - Kode fungsi
txtFunctionName VARCHAR(100) No - Nama fungsi
bitActive BIT No 1 Status aktif
txtCreatedBy VARCHAR(50) No - User pembuat
dtCreatedDate DATETIME No GETDATE() Tanggal dibuat
txtModifiedBy VARCHAR(50) Yes NULL User terakhir modifikasi
dtModifiedDate DATETIME Yes NULL Tanggal terakhir modifikasi

Indexes:

  • PRIMARY KEY: intBtpFunctionId
  • UNIQUE INDEX: txtFunctionCode
  • INDEX: bitActive

3.3 Table: M_BTP_FUNCTION_MAPPING

Purpose: Mapping relasi many-to-many antara BTP dan Function

Column Name Data Type Nullable Default Description
intMappingId INT No Auto Primary Key
intBtpId INT No - Foreign Key ke M_BTP
intBtpFunctionId INT No - Foreign Key ke M_BTP_FUNCTION
bitActive BIT No 1 Status aktif
txtCreatedBy VARCHAR(50) No - User pembuat
dtCreatedDate DATETIME No GETDATE() Tanggal dibuat
txtModifiedBy VARCHAR(50) Yes NULL User terakhir modifikasi
dtModifiedDate DATETIME Yes NULL Tanggal terakhir modifikasi

Indexes:

  • PRIMARY KEY: intMappingId
  • FOREIGN KEY: intBtpId REFERENCES M_BTP(intBtpId)
  • FOREIGN KEY: intBtpFunctionId REFERENCES M_BTP_FUNCTION(intBtpFunctionId)
  • UNIQUE INDEX: (intBtpId, intBtpFunctionId)
  • INDEX: bitActive

5. BUSINESS RULES

5.1 Master BTP Rules

  • Uniqueness: BTP Code harus unique dalam sistem
  • Format: BTP Code harus mengikuti format INS (International Numbering System)
  • Deletion: BTP yang sudah digunakan dalam formula tidak dapat dihapus (soft delete only)
  • Inactive: Inactive BTP tidak muncul dalam LOV untuk formula baru
  • Validation: BTP Code dan Name wajib diisi

5.2 Master BTP Function Rules

  • Uniqueness: Function Code harus unique
  • Format: Function Code harus uppercase, 3-5 characters
  • Standard: Function harus sesuai dengan regulasi BPOM
  • Deletion: Function yang sudah di-map ke BTP tidak dapat dihapus

5.3 BTP Function Mapping Rules

  • Many-to-Many: Satu BTP dapat memiliki multiple functions
  • No Duplicate: Tidak boleh duplicate function dalam satu BTP
  • Minimum: Setiap BTP harus memiliki minimal 1 function
  • Transaction: Bulk save menggunakan database transaction (all or nothing)
