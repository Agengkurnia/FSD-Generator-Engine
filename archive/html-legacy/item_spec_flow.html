🕰️ Sejarah Perkembangan Sistem

2018

Sistem Awal — 3 Modul Terpisah

Dibangun 3 form independen: Item Spec Blending (Prodev), Item Spec Packing (Packdev), dan Item Spec (NPC Team). Sistem ini berjalan baik untuk produk bertipe Base / Premix karena kedua tim bekerja secara independen.

⚠️
Masalah: Produk FG / Intermediate

Blending Spec dan Packing Spec saling bergantung — keduanya harus selesai sebelum Item Spec final bisa dibuat. Akibatnya terjadi bottleneck: saling tunggu-menunggu, dan tidak ada mekanisme "merge" data dari kedua spec ke satu dokumen final.

2019

Solusi — Modul Item Spec New

Lahir modul baru Item Spec New sebagai solusi bottleneck FG/Intermediate. Cara kerjanya: Prodev dan Packdev membuat spec-nya masing-masing secara independen. Setelah keduanya selesai (approved), NPC Team memilih kedua spec sebagai referensi, data langsung ter-link, lalu dikirim ke Oracle sebagai satu dokumen final.

✅
Keunggulan solusi ini

Prodev dan Packdev tidak perlu saling tunggu lagi. Masing-masing bisa mengerjakan spec-nya kapan saja — ketika keduanya selesai, Item Spec New tinggal link dan submit.

🧩 Detail Setiap Modul

📄

Item Spec

Era 2018 · Base/Premix
  • Dibuat oleh NPC / QC Team
  • Ada pilihan Inline / Non-Inline
  • Dikirim ke Oracle EBS
  • Tidak butuh referensi spec lain
  • Tidak ada IC Assignment
🧪

Item Spec Blending

Era 2018 · FG/Intermediate
  • Dibuat oleh Product Dev
  • Ada ProductCode, Variant, Country
  • IC Assignment (wajib saat submit)
  • Tidak dikirim ke Oracle
  • Approval: QA/QS internal
📦

Item Spec Packing

Era 2018 · FG/Intermediate
  • Dibuat oleh Packaging Dev
  • Struktur identik dengan Blending
  • IC Assignment (wajib saat submit)
  • Tidak dikirim ke Oracle
  • Approval: QA/QS internal
🏭

Item Spec New

Era 2019 · FG/Intermediate
  • Dibuat oleh NPC Team
  • Wajib pilih Blending Spec
  • Wajib pilih Packing Spec
  • Country ambil dari Blending Spec
  • Dikirim ke Oracle EBS

🚀 Rencana Revamp — IDC System

💡
Ide Utama

Semua modul Item Spec digabung menjadi 1 form tunggal di IDC System. User cukup pilih tipe transaksi di awal, form secara otomatis menampilkan field dan seksi yang relevan.

Perbedaan Tampilan Berdasarkan Tipe

🔵 Base / Premix

Field yang muncul: Input ingredient secara manual, atau opsi untuk copy dari Item Spec yang sudah ada (existing table). Tidak ada pilihan Blending/Packing Spec.

🟢 Intermediate / FG

Field yang muncul: Dropdown untuk Get data Blending Spec dan Get data Packing Spec. Data parameter uji otomatis ter-populate dari kedua spec yang dipilih.
